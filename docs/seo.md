# SEO architecture

> Last updated: 2026-04-17 · Owner: editorial / eng

Geck Inspect's SEO layer is cross-cutting and data-driven: every
content page is generated from structured data (morphs, care topics,
breeders), then prerendered at build time so non-JS crawlers see real
HTML, not the React SPA shell. This document is the map of that layer
— read it before changing anything that touches rendering, metadata,
or public routes.

## What's shipped

| Layer | Where | What it does |
|---|---|---|
| Route manifest | `scripts/seo-routes.mjs` | Single source of truth for indexable URLs. Expands morphs from `src/data/morph-guide.js` and care topics from `src/data/care-guide.js`. |
| Sitemap | `scripts/build-sitemap.mjs` → `public/sitemap.xml` | 97 URLs with `<lastmod>`, generated on `pnpm build`. |
| Prerender | `scripts/prerender.mjs` → `dist/<route>/index.html` | Per-route `<title>`, `<meta>`, canonical, OG, Twitter, plus a `<noscript>` body for non-JS crawlers (GPTBot, ClaudeBot, CCBot, PerplexityBot). Runs after `vite build`. |
| llms.txt | `public/llms.txt` | Navigation / welcome mat for AI crawlers. Hand-edited. |
| llms-full.txt | `scripts/build-llms-full.mjs` → `public/llms-full.txt` | Complete markdown corpus of care guide + morph catalog. Generated on every build. |
| IndexNow | `public/c9f2d53b0ac96f2ef77485a2b5d9b280.txt` | Bing / Yandex / Seznam push-indexing key. |
| Security + caching | `vercel.json` | HSTS, CSP headers, X-Frame-Options, Referrer-Policy, Permissions-Policy, long-lived asset caching, noindex X-Robots-Tag on admin surfaces. |
| Per-page SEO component | `src/components/seo/Seo.jsx` | Runtime metadata via react-helmet-async — overrides the static HTML for JS-executing crawlers. |
| Organization + WebSite JSON-LD | `src/lib/organization-schema.js` | Canonical Organization and WebSite schema, shared across pages. Also exports `breadcrumbSchema()`. |
| Editorial metadata | `src/lib/editorial.js` | `author`, `reviewedBy`, `datePublished`, `dateModified` for every content path. `bylineText()` renders the visible E-E-A-T byline. |
| Per-morph FAQPage | `src/lib/morphFaq.js` | Generates 3–6 Q&A pairs per morph from structured data; used for both visible FAQ section and FAQPage JSON-LD. |

or explicitly `noindex` via `vercel.json` X-Robots-Tag.

## Rendering contract

### Non-JS crawler path

1. Vercel filesystem serves `dist/<route>/index.html` for the exact
   requested path (generated by `scripts/prerender.mjs`).
2. The HTML contains route-specific `<title>`, `<meta description>`,
   `<link rel="canonical">`, OG + Twitter tags, and a visible
   `<noscript>` body with the page's substantive content and canonical
   URL.
3. Site-level Organization + WebSite JSON-LD remain in `index.html`
   and ship on every route.

### JS-executing crawler path

1. Same `dist/<route>/index.html` ships first.
2. React hydrates; `react-helmet-async` replaces `<title>`, meta tags,
   canonical, and injects per-page JSON-LD (Article, DefinedTerm,
   FAQPage, HowTo, BreadcrumbList, CollectionPage, ItemList).
3. No duplicate content — Helmet replaces in-place.

## Adding a new indexable URL

1. Add the route handler in `src/App.jsx` (public routes block) and
   in the authenticated routes if it should also have the app chrome.
2. On the page, render a `<Seo>` component with `title`, `description`,
   `path`, and `jsonLd`. See `src/pages/About.jsx` for the canonical
   shape.
3. Add the route to `STATIC_ROUTES` in `scripts/seo-routes.mjs` with
   `priority`, `changefreq`, `lastmod`, and optional `meta` for the
   prerenderer to use.
4. For programmatic URLs (e.g., a new data-driven hub), add a
   `getXxxRoutes()` function that expands the dataset into route
   objects, and include it in `getAllRoutes()`.
5. If the route has extractable text content, teach `prerender.mjs`
   how to build a useful `<noscript>` body for it — see `careTopicMeta`
   and `morphMeta` as patterns.

## Adding a new morph

No SEO changes required. Adding a `{ slug, name, summary, … }` entry
to `src/data/morph-guide.js` automatically:

- Lights up `/MorphGuide/<slug>` (served by `MorphDetail.jsx`).
- Expands the sitemap to include the new URL.
- Prerenders a route-specific HTML file with extracted title + summary.
- Appears on every taxonomy hub it matches.
- Gets a generated FAQPage and Article schema.
- Shows up in `llms-full.txt` on the next build.

## Adding a new care topic

Identical story for `src/data/care-guide.js`. Add a `{ id, title, body }`
section inside an existing category. On next build, the sitemap,
prerender, and `llms-full.txt` all pick it up.

## Checklist when editing existing content

- Bump `dateModified` in `src/lib/editorial.js` PER_PATH for the
  affected route (or category fallback) so the visible byline and JSON-LD
  stay honest.
- If the edit materially changes search intent (new topic coverage,
  new keywords), update the `description` on the `<Seo>` component
  and the `meta.description` in `scripts/seo-routes.mjs`.
- Re-run `pnpm build` and confirm the prerendered HTML looks right:
  `head -40 dist/<route>/index.html`.

## Non-goals, for clarity

- **We don't run SSR for authenticated, data-driven pages** (Marketplace,
  Gallery, Dashboard). They remain SPA-rendered; crawlers are allowed
  but not invited to those URLs beyond the sitemap mention.
- **We don't ship a Node render-to-string step** for React components.
  The prerender builds static HTML from structured data sources and
  injects a noscript block — this gives us the crawler signal we need
  without a bundler-aware SSR pipeline.
- **We don't maintain hand-written sitemap.xml**. Every URL in
  `public/sitemap.xml` is generated on `pnpm build` from
  `scripts/seo-routes.mjs`. Don't edit the XML directly — the file
  is overwritten on every build.

## When the audit comes back

The 2026-04 audit is tracked as the baseline. Items we considered out
of scope for the first pass:

- Populate `Organization.sameAs` with real social profiles (Instagram,
  YouTube, LinkedIn, TikTok, MorphMarket, Reddit, Wikidata) — the
  shape is in `src/lib/organization-schema.js`; add URLs to `SAME_AS`.
- Wire an IndexNow POST call into the deploy pipeline so content
  changes push to Bing / Yandex in near-real-time.
- Replace the editorial byline with specific Person entries (name,
  credentials, jobTitle) as named reviewers come on board.
- Expand `morph-guide.js` descriptions toward 400+ words per morph
  with citations to Repashy, de Vosjoli, Lilly Exotics, and Pangea.
